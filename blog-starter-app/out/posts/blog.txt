11:T1259,<h2>为什么选择 Next.js 搭建博客？</h2>
<p>Next.js 是由 Vercel 团队打造的 React 框架，具备以下优势：</p>
<ul>
<li>支持服务端渲染（SSR）与静态站点生成（SSG）</li>
<li>内置路由系统，零配置上手</li>
<li>拥有强大的社区模板和生态</li>
<li>非常适合构建 SEO 友好的博客系统</li>
</ul>
<p>而在开发过程中，借助 <strong>Cursor（AI 编程助手）</strong>，你可以高效写代码、调试组件甚至生成内容，大大提升开发体验。</p>
<hr>
<h2>一、使用官方模板快速启动</h2>
<p>我们选择使用 Vercel 提供的博客模板：<a href="https://github.com/vercel/next.js/tree/canary/examples/blog-starter">blog-starter</a></p>
<h3>✅ 项目初始化</h3>
<ol>
<li>
<p><strong>克隆模板项目</strong></p>
<pre><code class="language-bash">npx create-next-app my-blog -e https://github.com/vercel/next.js/tree/canary/examples/blog-starter
cd my-blog
</code></pre>
</li>
<li>
<p><strong>安装依赖</strong></p>
<pre><code class="language-bash">npm install
</code></pre>
</li>
<li>
<p><strong>本地启动</strong></p>
<pre><code class="language-bash">npm run dev
</code></pre>
<p>启动后访问：<code>http://localhost:3000</code>，即可预览初始博客页面。</p>
</li>
</ol>
<hr>
<h2>二、使用 Cursor 辅助开发（可选）</h2>
<p><a href="https://www.cursor.so/">Cursor</a> 是一个集成 GPT 和代码 IDE 的 AI 工具，强烈推荐用于：</p>
<ul>
<li>✨ 自动生成文章结构</li>
<li>🛠 快速构建组件 UI</li>
<li>📦 优化代码逻辑</li>
<li>🧠 调试 &#x26; 修复 bug</li>
</ul>
<p>只需在 Cursor 中打开项目，选择文件或使用 <code>/</code> 命令输入提示，例如：</p>
<pre><code>/rewrite this component using Tailwind CSS and make it mobile-friendly
</code></pre>
<hr>
<h2>三、自定义博客内容</h2>
<p>该模板使用 Markdown 格式存储博客内容，位于：</p>
<pre><code>/_posts/*.md
</code></pre>
<p>你可以根据模板格式撰写自己的博文，例如本文所用格式：</p>
<pre><code class="language-md">---
title: "你的文章标题"
excerpt: "简短介绍内容"
coverImage: "/assets/blog/your-image.jpg"
date: "2024-01-01"
author:
  name: "Wayne"
  picture: "/assets/blog/authors/headPic.jpg"
ogImage:
  url: "/assets/blog/your-image.jpg"
---
正文内容写在这里，支持 Markdown 格式。
</code></pre>
<hr>
<h2>四、自定义主题样式</h2>
<ul>
<li>
<p>模板使用 <code>Tailwind CSS</code>，你可以在 <code>styles/</code> 或 <code>components/</code> 中修改 UI 样式</p>
</li>
<li>
<p>替换默认头像和封面图：</p>
<ul>
<li><code>/public/assets/blog/introduce/html.jpg</code></li>
<li><code>/public/assets/blog/authors/headPic.jpg</code></li>
</ul>
</li>
</ul>
<hr>
<h2>五、部署博客到 Netlify（免费且快速）</h2>
<h3>📦 1. 初始化 Git 仓库并上传</h3>
<pre><code class="language-bash">git init
git remote add origin https://github.com/your-username/my-blog.git
git add .
git commit -m "init blog"
git push -u origin main
</code></pre>
<h3>☁️ 2. 登录 Netlify 并部署</h3>
<ol>
<li>
<p>访问 <a href="https://app.netlify.com/">https://app.netlify.com/</a></p>
</li>
<li>
<p>点击 “Add new site” → “Import from Git”</p>
</li>
<li>
<p>连接你的 GitHub 并选择仓库</p>
</li>
<li>
<p>设置构建配置：</p>
<ul>
<li><strong>Build command</strong>: <code>npm run build</code></li>
<li><strong>Publish directory</strong>: <code>out</code></li>
</ul>
</li>
<li>
<p>点击部署，几分钟后你将获得一个公开地址（形如 <code>https://your-blog.netlify.app</code>）</p>
</li>
</ol>
<h3>📝 注意：</h3>
<p>你需要在 <code>next.config.js</code> 中启用静态导出：</p>
<pre><code class="language-js">module.exports = {
  output: 'export',
}
</code></pre>
<p>这样执行 <code>npm run build</code> 时会生成 <code>out</code> 目录，供 Netlify 静态部署使用。</p>
<hr>
<h2>六、博客扩展方向</h2>

<li>🔍 增加搜索功能（使用 Algolia 或 FlexSearch）</li>
<li>💬 集成评论系统（如 giscus / utterances）</li>
<li>🧩 使用 MDX 增强 Markdown 支持</li>
<li>🔐 添加登录权限或管理员编辑界面</li>
</ul>
